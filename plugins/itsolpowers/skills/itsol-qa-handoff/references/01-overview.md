## Definition of Done

Definition of Done określa, kiedy praca może zostać uznana za zakończoną. Dotyczy standardu jakości zespołu, nie tylko jednej story.

Przykładowe DoD:

- kod spełnia acceptance criteria
- developer wykonał self-review
- testy automatyczne zostały dodane lub świadomie pominięte z uzasadnieniem
- testy lokalne przechodzą
- lint, formatowanie i type-check przechodzą
- migracje bazy są bezpieczne dla środowiska produkcyjnego
- logi i metryki są dodane tam, gdzie pomagają w utrzymaniu
- błędy są obsłużone i nie ujawniają danych wrażliwych
- code review zostało wykonane
- QA zweryfikowało zmianę albo zaakceptowało zakres testów automatycznych
- dokumentacja została zaktualizowana, jeśli zachowanie systemu lub API się zmieniło
- plan wdrożenia i rollbacku istnieje dla zmian ryzykownych
